Cleans and formats a subnetwork in the KN into a set of flat files for distribution.

Utilities for fetching and cleaning data in the KN mySQL database and then
formatting it into a set of files that can be easily distributed.  If the
output is not already present, queries the mysql database for a particular
subnetwork, cleans/normalizes the data using default recommendations, reformats
the data and metadata into .edge and .json files, and finally writes these
files to specificied directory ( including S3).

Arguments:
	edge_type (str): Limits edge types that are retrieved.
	species (int): The species taxon id to retrieve data for.
	edge_threshold (float): Limits edges based on weight.
	mysql host (str): mysql server credentials
	mysql_user (str): mysql server credentials
	mysql_port (int): mysql server credentials
	mysql_pass (str): mysql server credentials
	normalization/cleaning method (str): "default" using guidance by edge_type table 
	output_format (str): Specifies the output format. ".edge", ".df", etc
	output_location (str): Directory / S3 bucket to put the resulting files.
	output_credentials (str): S3 bucket access string

MySQL Queries:
	Edge File:
		SELECT s.edge_hash, s.n1_id, s.n2_id, s.weight, s.et_name, rl.file_id, rl.line_num FROM status s JOIN node_species n2 ON s.n2_id = n2.node_id JOIN raw_line rl ON s.line_hash = rl.line_hash WHERE s.et_name = 'pfam_prot' AND n2.taxon = 9606 AND s.status = 'production';
		SELECT s.edge_hash, s.n1_id, s.n2_id, s.weight, s.et_name, rl.file_id, rl.line_num FROM status s JOIN node_species n1 ON s.n1_id = n1.node_id JOIN node_species n2 ON s.n2_id = n2.node_id JOIN raw_line rl ON s.line_hash = rl.line_hash WHERE s.et_name = 'blastp_homology' AND n1.taxon = 9606 AND n2.taxon = 9606 AND s.status = 'production';
